# VMs Images

A Kubernetes Image is a container image that is deployed and managed within the cluster that includes everything needed to run a piece of software, including the code, runtime, libraries, environment variables, and configurations.

To access your project images, follow these steps:

<Steps>
  <Step title="First Step">
    At the Overview tab, select Projects.
  </Step>

  <Step title="Second Step">
    Choose your Project.
  </Step>

  <Step title="Third Step">
    Select VMs at your Project page.
  </Step>
</Steps>

<Note>Before creating a VM, you have to bind the image to the project first.</Note>

# Bind to Project

<Steps>
  <Step title="First Step">
    Click Images at the VMs page.
  </Step>

  <Step title="Second Step">
    You can access Bound Images, My Images and Public Images.
  </Step>

  <Step title="Third Step">
    After selecting an image you can bind it to the project by clicking on the Bind to Project button.
  </Step>
</Steps>

# Unbind Image
