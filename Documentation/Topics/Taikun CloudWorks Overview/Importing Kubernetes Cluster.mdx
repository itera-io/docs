---
title: "Importing Kubernetes Cluster"
---

By importing a Kubernetes cluster, users can use Taikun CloudWorks centralized management capabilities, streamlined resource allocation, and enhanced operational control.

This topic will guide you through importing your existing Kubernetes cluster into Taikun CloudWorks, empowering you to manage it effortlessly alongside your other cloud resources.

## What you’ll gain:

* Streamlined management of your existing Kubernetes cluster within Taikun CloudWorks.

* Centralized platform for all your cloud infrastructure, including Kubernetes.

* Simplified deployments, monitoring, and troubleshooting.

## 1. Prerequisites

Before importing your existing Kubernetes cluster into Taikun CloudWorks, it’s crucial to ensure that you have the necessary prerequisites.

**Access Credentials and Permissions:** Ensure you possess the required access credentials and permissions to import and manage the Kubernetes cluster. This includes administrator-level access to both the existing Kubernetes cluster and Taikun CloudWorks.

## 2. Create a new Taikun CloudWorks project by importing a kubeconfig

<Steps>
  <Step title="Step 1: Create a new Taikun CloudWorks project">
    Log in to the Taikun CloudWorks and locate the “Projects” section and press the “Add Project” button.
  </Step>

  <Step title="Step 2: Provide Cluster Details">
    Enter the credentials and details for the existing Kubernetes cluster, including the cluster’s address and authentication information.
  </Step>

  <Step title="Step 3: Obtain the kubeconfig File">
    Retrieve the kubeconfig file from the Kubernetes cluster. This file will establish a secure connection between Taikun CloudWorks and the imported cluster.
  </Step>

  <Step title="Step 4: Complete Import Process">
    Follow the on-screen prompts to finalize the import process. Taikun CloudWorks will establish a secure connection with the imported Kubernetes cluster, making it manageable from the Taikun CloudWorks platform.
  </Step>
</Steps>
